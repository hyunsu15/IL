# 테스트 원칙,워크 플로우

## 문서 설명
이 문서는 AI에게 시키기 위한 문서입니다.

## AI 확인사항

 - 초기 상태 + 각 Action의 최종 State만 검증 (중간 과정 ❌)
 - 결과값 비교는 Assertj 사용//자바 테스트 라이브러리
 - Thread.sleep 절대 금지
 - 피처플래그 분기 코드는 true/false 모두 테스트
 - 테스트 커버리지 70% 이상 달성
 - 테스트 작성 가이드를 반드시 읽고 작업
 - 품질 검토 체크리스트를 빌드 전에 반드시 수행

## 원칙

- 커밋 전 빌드 성공과 모든 테스트 통과를 필수로 검증합니다.
- 각 파일의 테스트 커버리지 70% 이상을 달성합니다.
- 테스트는 해당 모듈 내 test 폴더에 정확히 위치시킵니다.
- 프로덕션 소스코드는 절대 수정하지 않습니다.
- //은 주석이니 해당 내용은 무시한다.

## 워크 플로우

- guides폴더에 있는 자료를 읽고 테스트 관련 내용을 학습한다.

- REQUIREMENT.md에 정의되어 있는 요구사항을 분석한다.

- 학습했던 내용을 바탕으로 테스트 코드를 작성한다.

- 다시 guides폴더에 있는 자료를 읽고 작성했던 테스트 코드가 문서에 준수하도록 리팩토링한다.
