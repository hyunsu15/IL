# 테스트 작성 가이드

## Given-When-Then 패턴을 준수한다.

## 실패케이스,엣지 케이스, 예외상황을 포함시킨다.

## 만약 테스트코드에서 실패하는 상황은 FIXME 주석과 함께 테스트 통과하지 못하게 한다.
- ex)int add(int a,int b) => a=Integer.MAX_VALUE, b= 1 이면 오버플로우가 발생함. 그러므로 해당 값에 대한 테스트 코드는 FIXME 주석과 통과하지 못하게 만든다.

## 테스트코드는 조건문이 없어야한다.

## 실행할 때마다 결과가 달라지는 테스트를 만들면 안된다.

//java 관련
## Mockito 라이브러리를 이용하여 의존성 격리한다.
- 단위테스트는 내부 DB Mockito 사용

## 검증은 Assertj로 한다.
